## 发现的问题（待修复）

### 1. 测试失败：任务优先级变更导致测试失败
- **文件**：`src/group-queue.test.ts`
- **测试**：`drains tasks before messages for same group`
- **原因**：我们修改了策略，让用户消息优先于任务，但测试期望任务优先
- **影响**：一个测试失败，功能本身是合理的（用户消息应该优先）

### 2. CommonJS 语法问题
- **文件**：`src/container-runner.ts:171`
- **代码**：`const { execSync } = require('child_process');`
- **问题**：ESM 项目中使用 CommonJS 语法
- **风险**：虽然目前运行正常，但可能在某些环境下导致问题

### 3. Runtime API 无认证
- **文件**：`src/runtime-api.ts`
- **问题**：所有 API 端点都没有认证机制
- **风险**：任何能够访问 3456 端口的进程都可以调用所有 API
- **建议**：添加 API Key 认证或使用 Unix Socket

### 4. 输入验证不完整
- **文件**：`src/security.ts:84-103`
- **问题**：`validateUserInput` 仅检查原型链攻击模式
- **建议**：考虑添加更多验证类型（SQL 注入、XSS 等）

## 优化机会（待实施）

### 1. 性能优化

#### 1.1 数据库查询性能
- **文件**：`src/db.ts` 中的 `getNewMessages` 和 `getMessagesSince`
- **问题**：使用子查询和 ORDER BY，大型数据库下可能性能不佳
- **建议**：
  - 添加索引优化
  - 考虑分页查询
  - 缓存热点数据

#### 1.2 消息去重缓存性能
- **文件**：`src/index.ts` 中的 `isDuplicateMessage`
- **问题**：每次都清理过期缓存
- **建议**：
  - 使用 LRU 缓存策略
  - 周期性清理而非每次清理

#### 1.3 容器启动开销
- **问题**：每个消息都启动新容器
- **建议**：
  - 实现容器池化
  - 容器预热和复用

### 2. 代码质量优化

#### 2.1 错误处理完善
- **文件**：多个文件
- **问题**：部分异步操作没有错误处理
- **建议**：添加全面的错误处理，增强错误日志的详细程度

#### 2.2 代码去重
- **文件**：`src/db.ts` 等
- **问题**：`getMessagesSince` 和 `getNewMessages` 有大量重复
- **建议**：抽象公共逻辑，提高代码复用性

### 3. 安全加固

#### 3.1 Runtime API 认证
- **优先级**：高
- **建议**：添加简单的 API Key 认证

#### 3.2 IPC 目录权限
- **问题**：IPC 目录可能存在权限问题
- **建议**：确保目录权限配置正确
