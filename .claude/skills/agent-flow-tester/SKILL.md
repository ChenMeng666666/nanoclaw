# Agent Flow Tester

## 测试目标

全面测试NanoClaw Agent的完整流程，确保每个环节正常工作：

1. **Agent生命周期**：创建测试agent → 容器启动 → 发送消息 → 接收响应
2. **记忆管理**：验证记忆流转（L1→L2→L3）和定时固化L3任务
3. **学习系统**：创建学习计划 → 主动触发定时任务 → 检查学习记录
4. **进化系统**：经验上传 → 主动触发审核定时任务 → 验证进化库
5. **任务执行**：测试任务流程（搜索记忆→搜索进化库→外部学习→执行任务→经验上传）
6. **重复任务验证**：相同任务第二次执行应能应用进化库
7. **数据清理**：测试完成后清理所有测试数据
8. **系统组件**：验证定时任务、容器、数据库数据的完整性

## 测试环境要求

- Docker 服务正在运行（可选，用于容器测试）
- Node.js v18+ 已安装
- NanoClaw 项目已构建 (`npm run build`)

## 测试流程

## 测试脚本

### 1. 完整测试场景

使用 `tests/full-agent-flow.ts` 运行完整的 Agent 流程测试：

```bash
npm run test:full
```

这个测试覆盖了：

1. **基础 Agent 配置**
   - 创建测试 Agent
   - 验证数据库记录

2. **记忆流转测试**
   - 添加 L1 工作记忆
   - 添加 L2 短期记忆
   - 测试记忆检索
   - 验证记忆层级变化
   - **记忆固化测试**：主动触发 `consolidateMemoriesForAllAgents` 验证 L1→L2→L3 流转

3. **学习计划测试**
   - 创建学习任务
   - 验证任务创建
   - 检查任务状态

4. **进化库流转**
   - 提交经验
   - **立即触发审核**（5个代理审核）
   - 查询经验验证审核结果
   - 验证状态变更

5. **完整任务执行流程**
   - **真实任务模拟**：计算斐波那契数列
   - 搜索记忆系统（L1→L2→L3）
   - 搜索进化库
   - 外部学习（模拟）
   - 任务执行和验证
   - 经验总结和上传
   - 自动触发审核

6. **进化库重用测试**
   - 创建第二个 Agent
   - 查询进化库
   - 验证经验重用
   - 执行相同任务
   - 基于前人经验提交改进版经验

7. **定时任务触发**
   - 创建定时任务
   - 检查任务到期
   - 验证任务创建

8. **多Agent并发记忆系统**
   - 创建多个Agent同时执行任务
   - 验证记忆隔离性和并发访问
   - 测试并发记忆添加和检索
   - 验证各个Agent的记忆管理独立性

9. **进化库并发访问**
   - 多个Agent同时提交经验到进化库
   - 验证进化库审核系统的并发处理能力
   - 测试并发经验搜索和查询
   - 验证进化库数据一致性

10. **完整数据清理**
    - 清理所有测试数据
    - 验证数据库清理

### 2. 其他测试命令

```bash
# 最小化测试（快速验证）
npm run test:minimal

# 端到端测试（包含容器测试）
npm run test:e2e

# 运行所有测试
npm run test
```

## 测试结果评估

### 成功条件

- 所有测试用例通过
- 记忆管理正常工作
- 进化系统正常工作（经验上传→审核）
- 定时任务正确创建和管理
- 数据库记录完整
- 测试数据成功清理

### 失败条件

- 任何测试用例失败
- 记忆或经验未正确记录
- 数据未正确清理

## 注意事项

1. 测试过程会清理所有带 "test" 标记的数据
2. 容器相关测试需要 Docker 环境（已在测试中跳过）
3. 如遇到问题，查看日志输出进行调试
4. 建议在开发环境中运行测试，避免影响生产数据
