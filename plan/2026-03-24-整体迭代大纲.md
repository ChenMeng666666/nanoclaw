# 2026-03-24 整体迭代大纲

## 1. 目标说明

在蓝图尚未完全实现前，后续所有方案评估、架构取舍与模块设计，统一以这份整体蓝图为上位约束，避免局部最优破坏长期演进方向。

本轮先不进入编码，先明确整体迭代顺序、阶段依赖与每一阶段的输出目标。

## 2. 蓝图总览

### 2.1 五个核心方向

1. **Agent 独立运行单元**
   - 目标：实现 1 agent 1 容器
   - 要点：每个 agent 可独立配置自己的通道接入方式与模型参数

2. **Agent 私有记忆系统**
   - 目标：参考 mem0 与 qmd 的最佳实践，构建低 token 成本、高可检索、可理解关系的记忆体系
   - 要点：不仅保存事实，还要能表达关系、结构与图谱

3. **共同进化系统**
   - 目标：参考 EvoMap 思想，从日志、任务反思、短期记忆中提取经验，沉淀为可复用能力
   - 要点：避免重复踩坑，让 agent 在重复任务中持续变强

4. **Private Moltbook**
   - 目标：参考 Moltbook 思想，构建局域智能体社会
   - 要点：赋予 agent 社交工具、闲时触发能力，并与进化系统联动

5. **向上知识反哺**
   - 目标：把沉淀出的知识、规则、经验反哺主程序与 Claude Code 机制
   - 要点：形成从 agent 个体到系统母体的知识回流闭环

## 3. 排序原则

### 3.1 依赖优先

- 先做底座，再做中层能力，最后做系统级反哺
- 先做单体可闭环，再做多 agent 协同
- 先做可沉淀的数据结构，再做反思与进化
- 先做局域内部验证，再做向上整合

### 3.2 核心判断

- **1 是运行底座**：没有 agent 独立容器、独立配置，后续记忆、进化、社交都无法稳定绑定到明确主体
- **2 是认知底座**：没有可靠记忆，进化系统无法形成有效积累，社交系统也缺乏共享上下文
- **3 是能力增殖层**：建立在记忆之上，把任务经验转化为方法
- **4 是协同放大层**：建立在个体记忆与共同进化之上，把单 agent 能力扩展为局域社会能力
- **5 是系统级输出层**：应在前四项验证有效后，再向主程序和 Claude Code 机制沉淀

## 4. 推荐迭代顺序

## 阶段 0：蓝图治理与边界统一

### 目标

先统一术语、边界与评估标准，保证后续迭代不偏航。

### 本阶段应明确的内容

- agent、memory、reflection、evolution、social、upward flow 的统一定义
- 私有记忆、共享记忆、短期记忆、长期记忆的边界
- 单 agent 数据、群体数据、主程序数据的隔离边界
- 记忆写入、经验提炼、知识上行的触发条件
- 每阶段的验收方式

### 原因

如果没有阶段 0，后续每一项都可能在不同抽象层面上同时扩张，导致设计不断返工。

## 阶段 1：Agent 独立运行单元

### 对应蓝图

对应第 1 项：1 agent 1 容器，agent 可配置自己的通道和模型配置。

### 目标

把“agent 作为独立主体”这件事先立住。

### 关键输出

- agent 身份边界清晰
- 每个 agent 有独立运行容器
- 每个 agent 具备独立模型配置能力
- 每个 agent 具备独立通道绑定能力
- 后续记忆与进化能力都能绑定到明确 agent

### 为什么先做

后面所有系统都要回答一个问题：**这段记忆、这条反思、这次进化、这次社交行为到底属于哪个 agent。**

如果 agent 主体不先稳定，后面的所有沉淀都会变成混合状态，最终难以治理。

## 阶段 2：Agent 私有记忆系统

### 对应蓝图

对应第 2 项：基于 mem0 和 qmd 最佳实现构建记忆系统。

### 目标

为每个 agent 建立“低 token、可持续、可理解关系”的私有记忆能力。

### 关键输出

- 事实记忆
- 关系记忆
- 图谱或结构化关系表示
- 短期记忆与长期记忆的分层
- 检索与压缩机制
- 记忆写入、更新、遗忘、归档策略

### 为什么排在第二

- 进化系统需要从记忆系统读取结构化经验
- 社交系统需要以记忆作为个体上下文
- 向上知识反哺需要以记忆层为基础筛选高价值知识

### 本阶段重点

不是追求“记得更多”，而是追求“用更少 token 记住更有价值的内容，并保持关系可解释”。

## 阶段 3：共同进化系统

### 对应蓝图

对应第 3 项：基于 EvoMap 思想，从日志和任务反思中提取经验。

### 目标

让 agent 不只是保存记忆，而是能从任务实践中持续优化方法。

### 关键输出

- 日志提取机制
- 任务反思机制
- 短期经验抽取机制
- 坑点归纳机制
- 方法升级与经验复用机制
- 个体经验与群体经验的桥接机制

### 为什么排在第三

EvoMap 式进化需要输入材料，而这些输入材料主要来自：

- 已发生的任务日志
- 已沉淀的记忆
- 可比较的成功与失败经验

如果没有阶段 2，阶段 3 会停留在“生成反思文本”，但难以形成稳定的经验网络。

## 阶段 4：Private Moltbook

### 对应蓝图

对应第 4 项：基于 Moltbook 思想构建局域智能体社会。

### 目标

从“单体 agent 会记、会反思”升级到“多 agent 可协作、可交流、可在闲时演化”。

### 关键输出

- agent 社交工具
- agent 间互动协议
- 闲时触发机制
- 个体与群体进化联动机制
- 任务内协作与任务外交流能力

### 为什么排在第四

Private Moltbook 本质上放大的是已有 agent 能力：

- 没有独立 agent 主体，社会关系无从建立
- 没有稳定记忆，社交会退化成一次性对话
- 没有进化机制，闲时触发无法产生持续增益

所以它更适合作为“放大器”，而不是最先实现的层。

## 阶段 5：向上知识反哺

### 对应蓝图

对应第 5 项：向上知识反哺主程序 Claude Code 机制。

### 目标

把前面验证过的稳定知识、规则、经验与机制，反哺到更高层的主程序与通用工作流中。

### 关键输出

- 可上行的知识筛选标准
- 经验升级为规则的机制
- 反哺主程序的接口或流程
- Claude Code 机制可复用的通用增益

### 为什么最后做

向上反哺不应该反哺“草稿”，而应该反哺“已经被多轮任务验证过的稳定成果”。

如果过早上行，会把尚未成熟的局部设计固化进主程序，增加后续清理成本。

## 5. 推荐主线顺序

建议整体按以下主线推进：

1. **阶段 0：蓝图治理与边界统一**
2. **阶段 1：Agent 独立运行单元**
3. **阶段 2：Agent 私有记忆系统**
4. **阶段 3：共同进化系统**
5. **阶段 4：Private Moltbook**
6. **阶段 5：向上知识反哺**

可简化为一句话：

**先确定主体，再建立记忆，再形成进化，再扩展社会，再反哺母体。**

## 6. 当前建议的近期迭代节奏

如果下一步继续细化，建议优先拆成下面三个连续批次：

### 批次 A：把主体立住

- 先细化阶段 0 和阶段 1
- 先回答“agent 是谁、如何隔离、如何配置、如何绑定通道和模型”

### 批次 B：把记忆立住

- 再细化阶段 2
- 先回答“agent 记什么、怎么记、如何压缩、如何表达关系”

### 批次 C：把进化和社会立住

- 在主体与记忆稳定后，再细化阶段 3 和阶段 4
- 最后再定义阶段 5 的反哺边界

## 7. 本大纲的使用方式

- 在蓝图未完成前，所有新方案必须先判断自己属于哪一阶段
- 若某方案跨多个阶段，优先服务更靠前的依赖阶段
- 若某设计会破坏前置阶段边界，应视为暂缓项
- 后续版本可在 `plan/` 目录按日期持续追加迭代版本
