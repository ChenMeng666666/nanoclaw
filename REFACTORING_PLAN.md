This document outlines the Clean Architecture refactoring plan for the NanoClaw project. The goal is to decouple "God Objects" (>400 lines), separate concerns, and prepare the codebase for AI-driven V2.0 iteration.

## Phase 1: Foundation & Decoupling (P0)

Focus on extracting core domain services from mixed-responsibility helpers and managers.

- [x] **Extract Learning System Initialization**
    - **Source**: `src/application/bootstrap/message-orchestrator.ts` (lines 96-179)
    - **Destination**: `src/infrastructure/system/learning-system-initializer.ts`
    - **Goal**: Remove file system operations and template copying logic from the message orchestrator.

- [x] **Decompose Memory Manager Facade**
    - **Source**: `src/memory-manager.ts`
    - **Destination**:
        - `src/memory-manager/l1-cache-manager.ts` (Extract L1 caching logic)
        - `src/memory-manager/release-control-service.ts` (Extract release control & rollback logic)
    - **Goal**: Reduce `MemoryManager` to a true facade that delegates to focused sub-components.

- [x] **Split Learning Helpers into Domain Services**
    - **Source**: `src/runtime-api-learning-helpers.ts`
    - **Destination**:
        - `src/domain/learning/services/learning-needs-analyzer.ts` (Extract `analyzeLearningNeeds`, `normalizeLearningNeeds`)
        - `src/domain/learning/services/learning-outcome-analyzer.ts` (Extract `analyzeLearningOutcome`, `extractKnowledgePoints`)
        - `src/domain/learning/services/reflection-generator.ts` (Extract `generateRuntimeReflection`, `generateDailySummary`)
    - **Goal**: Eliminate the "helpers" dumping ground and create proper Domain Services.

## Phase 2: Evolution System Refactoring (P1)

Focus on breaking down the monolithic Evolution Manager and Repository.

- [x] **Decompose Evolution Manager**
    - **Source**: `src/evolution-manager.ts`
    - **Destination**:
        - `src/domain/evolution/services/evolution-service.ts` (Core GEP protocol logic, `submitExperience`, `submitGene`)
        - `src/domain/evolution/services/capsule-service.ts` (Capsule creation, promotion logic, `createCapsule`)
        - `src/domain/evolution/services/strategy-service.ts` (Strategy configuration and management)
    - **Goal**: Reduce `EvolutionManager` size (<300 lines) and separate GEP protocol concerns.

- [x] **Refactor Evolution Repository**
    - **Source**: `src/infrastructure/persistence/repositories/agent/evolution-repository.ts`
    - **Destination**:
        - `src/infrastructure/persistence/repositories/agent/evolution-read-repository.ts` (Read operations, `getEvolutionEntry`, etc.)

    - **Goal**: Segregate Command and Query responsibilities (CQRS preparation) for the database layer.

## Phase 3: Learning & Reflection Refactoring (P1)

Focus on separating scheduling mechanics from business logic execution.

- [x] **Separate Reflection Scheduling from Execution**
    - **Source**: `src/reflection-scheduler.ts`
    - **Destination**:
        - `src/application/scheduling/reflection-scheduler.ts` (Keep only cron triggering logic)
        - `src/application/learning/reflection-executor.ts` (Move `triggerReflection`, `completeLearningTask`, content generation orchestration)
    - **Goal**: The Scheduler should only know *when* to run, not *how* to run the business logic.

- [x] **Modularize Learning HTTP Handlers**
    - **Source**: `src/interfaces/http/handlers/learning-legacy-handlers.ts`
    - **Destination**:
        - `src/interfaces/http/handlers/learning/automation-handlers.ts` (Start/Stop/Status)
        - `src/interfaces/http/handlers/learning/planning-handlers.ts` (Generate Plan, Analyze Needs)
        - `src/interfaces/http/handlers/learning/analysis-handlers.ts` (Analyze Outcome, Extract Knowledge)
    - **Goal**: Split the giant handler file into feature-specific handlers that call the new Domain Services from Phase 1.

    - **Destination**: `src/application/bootstrap/bootstrap.ts`
    - **Goal**: Move Channel setup loop, Subsystem initialization (Scheduler, IPC), and detailed error handling into a `Bootstrap` class. `index.ts` should only call `Bootstrap.init()`.

- [ ] **Externalize Signal Patterns**
