# 学习体系变更门禁（P3）

## 1. 适用变更

以下变更必须走本门禁：
- 学习调度语义变更
- 反思频率与生成策略变更
- Runtime API 鉴权与参数契约变更
- 学习模型分层与降级策略变更
- 学习结果入记忆与进化策略变更

## 2. 门禁分层

### 2.1 契约门禁（必过）

必须通过：
- `src/learning-p0-contract.test.ts`
- `src/learning-p1-contract.test.ts`
- `src/learning-p2-contract.test.ts`

失败即阻断提交。

### 2.2 集成门禁（必过）

必须通过仓库统一门禁：
- `npm run typecheck`
- `npm run test`

失败即阻断合并。

### 2.3 语义门禁（必过）

满足以下条件方可合并：
- 变更说明明确影响范围。
- 文档已同步：审计计划、运维手册、验收矩阵。
- 验收矩阵中受影响场景已复验并更新状态。

### 2.4 发布门禁（必过）

满足以下条件方可进入发布分支：
- 契约门禁与集成门禁全部通过。
- 治理清单对应项已完成并有证据。
- 回滚策略可执行且已完成至少一次演练记录。

## 3. 标准提交流程

1. 实施变更并补充文档。
2. 执行契约门禁与集成门禁。
3. 更新验收矩阵中受影响行。
4. 发起评审并附验证结果。
5. 审查通过后进入发布流程。

## 4. 评审检查清单

- [ ] 是否存在未覆盖的契约变更
- [ ] 是否遗漏调度/鉴权/模型策略的语义说明
- [ ] 是否更新运维手册与验收矩阵
- [ ] 是否保留失败原因码与降级证据
- [ ] 是否具备可执行回滚路径

## 5. 不合规处理

- 未通过契约门禁：禁止合并，必须先修复。
- 文档或矩阵未更新：评审直接驳回。
- 缺少回滚路径：禁止进入发布分支。
