# 学习体系端到端验收矩阵（P3）

## 1. 目标

- 将“用户学习指令 -> 调度 -> 执行 -> 反思 -> 记忆 -> 进化”固化为可重复验收基线。
- 为回归测试和发布评审提供统一判定标准。

## 2. 验收矩阵

| ID | 场景 | 输入 | 期望行为 | 关键验证点 | 结果 |
| --- | --- | --- | --- | --- | --- |
| E2E-01 | 标准学习流 | 用户指定学习主题 + 每天固定时间 | 创建/复用反思任务，生成计划并创建合法调度 | 计划创建成功；调度存在；任务执行可追踪 | [x] |
| E2E-02 | 间隔学习流 | 用户指定学习主题 + 每隔多久 | 时间偏好映射到标准调度表达 | 调度表达合法；下次执行可计算 | [x] |
| E2E-03 | 已存在反思任务 | 用户重复下达同方向学习 | 复用现有反思任务，不重复创建 | 任务去重生效；链路继续执行 | [x] |
| E2E-04 | 鉴权失败 | API key 缺失或错误 | 请求被拒绝且链路可观测 | 鉴权失败信号清晰；无脏数据写入 | [x] |
| E2E-05 | 调度非法 | 输入无效时间表达 | 计划拒绝落库并返回明确原因 | 原因码明确；无错误调度创建 | [x] |
| E2E-06 | 模型降级 | SDK 超时或失败 | 自动降级到兜底路径并记录原因 | `degrade_reason` 可追踪；主流程不中断 | [x] |
| E2E-07 | 执行超时 | 学习任务执行超时 | 状态可恢复且支持重试策略 | 超时告警可见；状态流转一致 | [x] |
| E2E-08 | 反思到记忆闭环 | 正常完成学习任务 | 反思产出并进入记忆层 | 反思记录存在；记忆写入成功 | [x] |
| E2E-09 | 记忆到进化闭环 | 高价值学习结果 | 触发进化候选提交 | 进化候选记录可审计 | [x] |
| E2E-10 | 回归一致性 | 同类输入重复执行 | 输出行为稳定，不依赖偶然触发 | 链路关键节点一致 | [x] |

## 3. 执行基线

- 契约测试基线：
  - `src/learning-p0-contract.test.ts`
  - `src/learning-p1-contract.test.ts`
  - `src/learning-p2-contract.test.ts`
- 必须在提交前执行并通过。

## 4. 结果判定规则

- 全部核心场景通过，方可判定“P3 验收通过”。
- 任一核心场景失败，必须补充修复与复验记录。
- 每次发布前至少复验 E2E-01、E2E-04、E2E-06、E2E-08、E2E-10。

## 5. 维护规则

- 新增学习链路能力时，必须追加矩阵行并定义验证点。
- 变更调度、反思、鉴权、模型策略时，必须复验受影响场景。
- 每季度治理评审时同步更新本矩阵状态。
