# 学习体系运维手册（P3）

## 1. 适用范围

- 适用于学习体系主链路：学习意图编排 -> 计划与调度 -> 执行 -> 反思 -> 记忆 -> 进化。
- 适用于主进程调度单一真相源（SSOT）模式。
- 适用于 Runtime API 开启鉴权场景。

## 2. 快速健康检查

```bash
# 1) 核心服务是否正常
launchctl list | grep nanoclaw

# 2) 学习契约测试基线是否通过
npm run test -- src/learning-p0-contract.test.ts src/learning-p1-contract.test.ts src/learning-p2-contract.test.ts

# 3) 关键日志信号
grep -E 'learning|reflection|schedule|degrade|reasonCode' logs/nanoclaw.log | tail -50
```

通过标准：
- 契约测试通过。
- 无连续失败告警。
- 关键链路日志存在成功信号。

## 3. 六段式排障路径

### 3.1 学习意图识别

检查点：
- 用户学习方向是否进入学习工作流。
- 学习主题、节奏字段是否被提取。

常见异常：
- 意图未识别。
- 主题为空或语义过弱。

处置：
- 复核入口识别规则与最近输入样本。
- 必要时补充更明确的学习指令模板。

### 3.2 计划创建

检查点：
- 计划结构完整。
- 时间表达合法。

常见异常：
- 计划生成成功但字段缺失。
- 时间偏好未映射到有效调度。

处置：
- 回看计划生成日志与返回负载。
- 校验字段与契约测试中的预期一致。

### 3.3 调度落库

检查点：
- 调度任务写入成功。
- 下次执行时间可计算。

常见异常：
- 调度未创建。
- 调度类型不被识别。

处置：
- 检查调度创建日志与失败原因码。
- 回归验证调度表达与契约一致。

### 3.4 执行状态

检查点：
- 任务 start/complete 状态成对出现。
- 超时与重试行为可观测。

常见异常：
- 启动成功但未完成。
- 连续重试无收敛。

处置：
- 检查任务状态流转日志。
- 对超时场景执行降级策略并保留原因码。

### 3.5 反思产出

检查点：
- 反思生成成功。
- 降级路径可追踪。

常见异常：
- 反思为空。
- 高价值反思未走预期模型策略。

处置：
- 检查 `degrade_reason` 与模型选择日志。
- 复核质量门禁指标与抽样结果。

### 3.6 记忆与进化入库

检查点：
- 记忆写入成功。
- 进化候选提交成功或有明确拒绝理由。

常见异常：
- 记忆写入失败。
- 进化提交失败且无可解释原因。

处置：
- 检查入库链路日志与失败原因码。
- 触发恢复流程后重放单条样本验证闭环。

## 4. 观测指标与告警建议

核心指标：
- 计划可执行率
- 反思有效率
- 降级触发率
- 端到端时延
- 失败原因码分布

告警建议：
- 连续失败超过阈值触发告警。
- 降级触发率异常上升触发告警。
- 端到端时延超过上限触发告警。

## 5. 故障演练清单

每月至少完成一次，至少覆盖以下场景：
- 鉴权失败
- 调度表达非法
- SDK 超时降级
- 记忆写入失败
- 进化提交失败

演练记录必须包含：
- 触发条件
- 观测信号
- 处置动作
- 回滚动作
- 复盘结论

## 6. 回滚策略

优先级：
1. 回滚策略与配置
2. 回滚流程开关
3. 回滚代码版本

执行要求：
- 回滚前后均记录关键指标快照。
- 回滚完成后必须复跑学习契约测试。
- 回滚结论进入季度治理评审输入。
